		Interview Answer for JVM

JVM stands for Java Virtual Machine. It is responsible for converting Java bytecode into machine code and executing it. It provides platform independence because bytecode can run on any operating system having JVM installed. JVM also handles memory management, garbage collection, class loading, and runtime execution.



Interview Answer for JRE

JRE stands for Java Runtime Environment. It provides the runtime environment required to execute Java applications. JRE includes JVM, Java libraries, and supporting files but does not contain development tools like compiler. It is mainly used when users only need to run Java applications.


Interview Answer for JDK

JDK stands for Java Development Kit. It is a complete package used for developing Java applications. It contains JRE along with development tools such as compiler, debugger, documentation tools, and packaging utilities. Developers use JDK to write, compile, test, and deploy Java applications.


Short Interview Summary (2-Minute Answer)

JVM is the execution engine that converts Java bytecode into machine code and provides memory management and garbage collection. JRE provides the runtime environment required to run Java applications and contains JVM plus libraries. JDK is the complete development kit containing JRE and tools like compiler, debugger, and packaging utilities. In short, JDK is used for development, JRE for running applications, and JVM for executing bytecode.

Java 8 was a major release that introduced functional programming concepts. Important features include Lambda Expressions for reducing boilerplate code, Stream API for collection processing, Functional Interfaces to support lambdas, Optional for null handling, new Date-Time API for thread-safe date operations, Method References, Default and Static methods in interfaces, Parallel Streams, and CompletableFuture for asynchronous programming. Functional Interface in Java 8 is an interface that contains exactly one abstract method and is mainly introduced to support lambda expressions and functional programming. We can annotate it with @FunctionalInterface to enforce compile-time checking. It can also contain multiple default and static methods. Common examples are Predicate, Consumer, Supplier, and Function. In projects, I have used functional interfaces extensively with streams and lambda expressions to reduce boilerplate code.

Why Default Methods are Used in Java 8?
Problem Before Java 8:

Suppose you have an interface:

interface Vehicle {
    void start();
}

Many classes implement it:

class Car implements Vehicle {
    public void start() {
        System.out.println("Car Started");
    }
}

Now imagine after 5 years you want to add a new method:

interface Vehicle {
    void start();
    void stop();   // newly added
}

Problem:

Every implementing class breaks
All classes must implement stop()
Existing projects become incompatible
Solution: Default Method
interface Vehicle {

    void start();

    default void stop() {
        System.out.println("Vehicle stopped");
    }
}

Now existing classes continue working because interface provides implementation.

Interview Answer:

"Default methods were introduced in Java 8 mainly for backward compatibility. They allow adding new functionality to interfaces without breaking existing implementations. They are also heavily used in Collections API and Streams."

Example:
interface Animal {

    void sound();

    default void sleep() {
        System.out.println("Sleeping");
    }
}

class Dog implements Animal {

    public void sound() {
        System.out.println("Bark");
    }
}

Usage:

Dog d = new Dog();

d.sound();
d.sleep();
Why Static Methods are Used in Interfaces?

Static methods are added when a method:

Belongs to interface itself
Does not depend on object state
Provides utility/helper functionality
Example:
interface Calculator {

    static int add(int a, int b) {
        return a + b;
    }
}

Call:

System.out.println(Calculator.add(10,20));

Output:

30
Why not default?

Default methods require object creation:

obj.method()

Static methods:

InterfaceName.method()

No object required.

Real Reason Java Added Static Methods

Before Java 8, utility classes were used:

Collections.sort(list);

Java 8 added many static methods inside interfaces themselves:

Example:

List.of()
Comparator.comparing()
Map.entry()

This keeps related utilities inside the interface.
