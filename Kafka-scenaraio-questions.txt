Interview Answer

“KafkaTemplate is a Spring Kafka helper class used to simplify producing messages to Kafka topics. It abstracts producer creation, connection handling, serialization, and message publishing. In my payment microservice, I used KafkaTemplate to publish OrderEvent objects asynchronously to the order-events topic, which are then consumed by downstream services.”

How Kafka Partition Gets Selected (Brief Interview Answer)

If Key is Provided:

kafkaTemplate.send("order-events", "101", event);

Kafka calculates:

Partition = hash(key) % number_of_partitions

Example:

Key = "101"
hash(101) % 3 partitions = Partition 1

Why use key?

Same key always goes to same partition
Maintains message ordering for that key
Useful for orderId, customerId, userId

If Key is NOT Provided:

kafkaTemplate.send("order-events", event);

Kafka producer chooses partition automatically using its default partitioner.

Usually messages are distributed across partitions to balance load.

Interview Ready Answer

“Partition selection in Kafka depends on whether a key is provided. If a key exists, Kafka uses hash(key) % number_of_partitions to decide the partition so messages with the same key maintain ordering. If no key is provided, Kafka automatically distributes messages across partitions for load balancing.”
